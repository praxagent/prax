# TeamWork vs. Microsoft Loop → moved

This comparison now lives in the **TeamWork repo**, where TeamWork's own docs
belong: `teamwork/docs/comparisons/microsoft-loop.md`.

It's kept in this research index because it's part of Prax's landscape-analysis
series, but the canonical copy is TeamWork-side (TeamWork is agent-agnostic — its
product docs serve non-Prax users too).

**One-line takeaway:** Loop and TeamWork rhyme on the noun ("a workspace for a
team") but diverge on the verb — **Loop is for humans co-creating content
(Copilot assists); TeamWork is for humans collaborating with AI agents that do
work (you supervise).** Not competitors. The single transferable idea is Loop's
**portable, live-syncing component**, tracked as a Prax↔TeamWork cross-cutting
item in [`../IDEAS_BACKLOG.md`](../IDEAS_BACKLOG.md) #21 (and TeamWork's own
backlog).
